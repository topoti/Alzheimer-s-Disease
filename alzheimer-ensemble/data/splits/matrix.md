# Split matrix — how a filename maps to its partition

Generated by `src/split_subjects.py`. Two CSVs, same split at different grain:
`subject_split_3class.csv` (1 row/patient) and `slice_index_3class.csv` (1 row/image).

## filename → split
```
OAS1_0351_MR1_mpr-4_160.jpg
└── OAS1_0351 = subject ID  → decides the partition (all its slices go together)
    MR1 session · mpr-4 acquisition · 160 slice index (not used for splitting)
```
The `OAS1_XXXX` ID is the grouping key. Splitting by subject (never by image)
is what prevents slice leakage.

## fold column (the whole partition lives here)
| fold | meaning |
|------|---------|
| `-1` | held-out **test** set — touched once, at the very end |
| `0`–`4` | the 5 **CV folds** — on run *k*, fold *k* is validation, the other 4 train |

No separate dev/test column: `fold == -1` is test, `fold >= 0` is dev.

## class groups (original folder → label)
| orig_class (folder) | label | class_name |
|---------------------|-------|------------|
| Non Demented        | 0 | no_dementia |
| Very mild Dementia  | 1 | very_mild |
| Mild Dementia       | 2 | mild_moderate |
| Moderate Dementia   | 2 | mild_moderate |

Moderate (only 2 subjects) is merged into Mild → **3-class task**.
`label` is stored; `class_name` is dropped (1:1 with label). `orig_class` is kept
because it is the only trace of which label-2 rows were originally Moderate.

Total: 347 subjects · 86,437 images (266 / 58 / 23 per class).
